# tools 的功能

`tools/` 用来放管理这套共享 skills 的脚本。

这里的脚本不是 skill 本体，而是用来接入、检查、创建、打包和维护 skill 库。

## 常用脚本

- `Connect-Skills.ps1`：把 `library/skills/` 接入到本机 `.claude/skills`、`.codex/skills` 和 `.agents/skills`。
- `Audit-Skills.ps1`：检查 `library/skills/` 是否都是标准 skill，并生成 `docs/SKILL_INDEX.md`。
- `Audit-SkillTriggers.ps1`：检查 skill description 是否适合精准触发，并生成 `docs/TRIGGER_AUDIT.md`。
- `Test-SkillRouting.py`：用 `docs/trigger-evals.json` 做跨平台代理路由测试。
- `Invoke-SkillQualityGate.ps1`：一键运行结构审计、触发审计、路由测试、manifest 导出、真实平台结果汇总和默认接入目录检查；`-RequireRealPlatformComplete` 会要求真实平台结果全为 100%。
- `Export-SkillManifest.ps1`：导出 `docs/skills-manifest.json` 和 `docs/SKILLS_MANIFEST.md`，供通用 agents 读取。
- `Export-PlatformValidationPack.ps1`：导出真实 app 手动验证矩阵和 CSV 提示表。
- `Summarize-PlatformValidation.ps1`：汇总真实 app 手动验证结果，生成 `docs/REAL_PLATFORM_VALIDATION_SUMMARY.md`。
- `创建新技能.py`：创建新 skill 的辅助脚本。
- `打包技能.py`：打包 skill 的辅助脚本。

## 常用命令

```powershell
.\tools\Invoke-SkillQualityGate.ps1
.\tools\Audit-Skills.ps1
.\tools\Audit-SkillTriggers.ps1
python .\tools\Test-SkillRouting.py --evals .\docs\trigger-evals.json --skills .\library\skills --out-dir .\docs
.\tools\Export-SkillManifest.ps1
.\tools\Export-PlatformValidationPack.ps1
.\tools\Summarize-PlatformValidation.ps1
.\tools\Connect-Skills.ps1 -Targets All
```

## 维护规则

- 不要把普通资料、压缩包或 skill 本体放在这里。
- 修改脚本后，至少运行一次 `Audit-Skills.ps1` 验证目录状态。
