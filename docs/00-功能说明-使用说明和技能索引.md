# docs 的功能

`docs/` 用来放这个共享技能库的说明文档和自动生成索引。

这里的文件主要回答三类问题：

- 怎么在新电脑接入这套 skills。
- 怎么查看当前有哪些标准 skills。
- 后续怎么新增、整理、排障。

## 重要文件

- `ACCESS_GUIDE.md`：多电脑接入指南。
- `SKILL_INDEX.md`：自动生成的 skill 索引，由 `tools/Audit-Skills.ps1` 更新。
- `TRIGGER_AUDIT.md`：自动生成的触发词静态审计报告。
- `TRIGGER_TEST_RESULTS.md`：自动生成的跨平台代理路由测试报告。
- `skills-manifest.json`：给不支持目录扫描的通用 agents 使用的机器可读清单。
- `SKILLS_MANIFEST.md`：manifest 的人类可读版本。
- `REAL_PLATFORM_VALIDATION_MATRIX.md`：Claude Code、Codex、Trae/SOLO、豆包/Marvis/GLM 等真实 app 的手动验证矩阵。
- `REAL_PLATFORM_VALIDATION_SUMMARY.md`：真实 app 手动验证结果汇总。
- `platform-prompt-packs/`：每个平台一份的 65 条实测提示包。
- `QUALITY_GATE_REPORT.md`：一键质量闸门的最新运行结果。

## 维护规则

- 新增或移动标准 skill 后，运行 `.\tools\Audit-Skills.ps1` 更新 `SKILL_INDEX.md`。
- 修改触发描述或新增 skill 后，运行触发审计、路由测试、manifest 导出和平台验证包导出。
- 手写说明文档可以放在这里。
- 不要把 skill 本体放到这里；标准 skill 应放到 `library/skills/`。
